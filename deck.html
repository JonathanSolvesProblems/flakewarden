Business Impact & Adoption
The problem

Flaky tests are the most corrosive failure mode in CI.

When a build goes red, you often can't tell a real regression from noise.

~16%
of Google's tests showed some flakiness
~84%
of pass→fail transitions came from flaky tests
~100
spurious failures per run at 5% flake on a 2,000-test suite
The real cost isn't the triage minutes. Teams stop trusting red builds, start ignoring them, and a genuine regression ships to production.

Creativity & Innovation
The idea

Answer the one question that matters, then route the failure.

real_defect → escalate flaky → propose a fix (human-gated) environment → re-run on clean infra

The uniqueness claim

Detection tools tell you a test is flaky. Healing tools auto-repair selectors. FlakeWarden is the governance layer between them: a reproducible, auditable scorer that spends an LLM only on the ambiguous band, under a hard safety contract (a real regression is never auto-quarantined as flaky), with every fix a human-gated proposal written back through UiPath, not an autonomous mutation.

The moat is the composition + safety contract, not any single component. Positioned vs Datadog / Develocity / Trunk (detection) and Healenium / Tricentis / UiPath Autopilot (healing) in docs/prior-art.md.

AI Factor · Creativity
The thesis

Deterministic where the math must be exact. Generative where the evidence is messy.

◧ Deterministic flake-scorer

  • Auditable statistics over run history, no model call
  • Resolves the confident cases and never guesses
  • 52 / 150 failures resolved here, ~⅓, zero LLM spent
  • Reproducible: same input → same verdict, every time

◍ Grounded Agent Builder classifier

  • RAG over stack trace, DOM diff, commit message, runner logs
  • Reasons over only the ambiguous band (the other 98)
  • Cites the evidence; tie-breaks toward real_defect
  • Runs on UiPath AI Trust Layer (PII redaction + audit)
And a human approves every change. The model proposes; it never mutates the suite on its own.
The deterministic-vs-generative boundary is the headline AI design decision4 / 12

Business Impact · Governance
Governance & the human gate

The agent informs the decision. The human makes it.

  • No autonomous mutation. Every heal, quarantine, or baseline change is a proposal → Action Center.
  • Fail-safe default. On low confidence the agent frames the failure as a defect, never silently quarantines.
  • Eval-driven release gate. The agent can't publish below the accuracy + safety threshold.
  • Negative control. 0 real defects hidden and 0 defects auto-healed across all 150 cases.
This is the line a QA lead has to trust: the system escalates when it's unsure, instead of hiding.
Aligns with "information, not automation": AI works with you, not instead of you9 / 12

Feasibility · Completeness
Honest limitations & path to production

What's real today, and what production needs.

Real today

  • Live grounded agent classifying all three classes on UiPath
  • Maestro orchestration running end-to-end, Successful
  • Measured accuracy + a mechanism-enforced 0% safety rate
  • Production-shaped architecture, governance, and eval methodology

The gap is the data

  • Corpus is synthetic-but-adversarial: a solo builder can't ship a real enterprise's CI history
  • Production needs the live Test Manager results API + gold-standard labels
  • Then a shadow-mode prospective study to report real-world accuracy
  • Drift monitoring is scaffolded, not solved (named honestly)

The architecture, governance gates, and eval methodology are production-shaped. The data is the honest gap.
